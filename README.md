# QPet

一个基于 **Rust + Axum + Spring Boot** 构建的可扩展桌宠平台，支持 **Mod/角色商城分发、运行时热切换、Persona 驱动的 AI 对话**。

## 项目简介

QPet 不只是一个桌宠应用，而是一个由 **桌面客户端、本地网关服务、商城服务** 组成的桌面系统：

- **桌面客户端（Rust）**：负责桌宠渲染、动画、交互、菜单和聊天 UI
- **本地网关（Rust / Axum）**：负责角色管理、资源下载、解压安装、Persona 同步、AI 对话、记忆与日志等能力
- **商城服务（Java / Spring Boot）**：负责角色/Mod 展示、购买、下载与数据管理

系统支持通过商城下载角色资源包，并在本地完成安装、扫描与热切换；同时每个角色都带有独立的 `character.json` 人设配置，可驱动符合角色设定的 AI 对话体验。

---

## 核心特性

- **Mod/角色分发**
  - 支持通过商城下载角色或资源包
  - 支持 ZIP 资源安装与本地解压
  - 支持运行时热切换，无需重启客户端

- **Persona 驱动 AI 对话**
  - 每个角色由 `character.json` 定义人格、背景、语气风格和台词
  - 切换角色后自动同步当前 Persona
  - 聊天请求会将 Persona 注入 system prompt，使 AI 回复符合角色设定

- **本地网关架构**
  - 通过本地网关统一管理角色、安装、聊天、日志与系统信息
  - 解耦桌面端与业务逻辑，便于后续扩展更多服务能力

- **实时通信**
  - 桌面端与网关通过 WebSocket 通信
  - 支持 Persona 同步、互动事件上报、自动对话推送

- **安全与稳健性**
  - ZIP 解压限制（entry 数、文件大小、总量限制）
  - 文件访问路径白名单
  - 前端渲染内容转义，降低 XSS 风险
