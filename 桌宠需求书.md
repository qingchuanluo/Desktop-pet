1. 项目简介

AI 桌宠系统（Syspet）是一个运行于 Windows 桌面的轻量级智能 Agent，由两部分组成：

桌宠前端：展示动画、渲染、用户交互

AI 后台 Daemon：人格系统、长期记忆、系统监控、自动日记

系统功能包括：

桌面透明窗口展示角色

PNG 动画渲染

点击、拖拽、互动行为

后台运行的 AI 人格

文件投喂、行为感知、自动写日记

持续学习用户习惯

不依赖 GPU，可在无显卡设备上稳定运行。

2. 功能需求
2.1 前台桌宠（Softbuffer CPU 渲染）
功能	描述
透明窗口	无边框、置顶、可穿透
PNG 动画	多动作帧动画、自动播放
位置拖动	桌宠可被拖动、记录位置
点击响应	播放表情、台词、动作
悬浮行为	鼠标靠近改变状态
自动动作	后台人格驱动 idle 触发
2.2 AI 后台（Daemon）
模块	描述
人格系统	固定个性 + 可成长型长期记忆
记忆库	SQLite / JSON 长期存档
文件投喂	拖进桌宠窗口的文件会被 AI 总结
AI 日记	每日记录用户行为、宠物状态、情绪
事件监控	CPU、焦点窗口、文件变化等
自治行为	无交互时自动“自言自语/做动作”
2.3 双进程通信

通信方式：

TCP 127.0.0.1

或 Windows Named Pipe

JSON 消息格式

示例：

{
  "event": "click",
  "time": "2026-03-13",
  "payload": {}
}
2.4 设置面板（可选）

桌宠外观切换

个性配置

记忆管理

行为日志查看

AI 对话窗口

3. 非功能性需求
3.1 性能指标

CPU 占用：空闲 < 5%，动画 < 10%

内存占用：< 200MB

AI 处理延迟：< 200ms（本地模型）

3.2 兼容性

Windows 10/11

无需 GPU 驱动

不需要任何依赖环境（打包后即可运行）

Rust（桌面 + 后台）

serde_json

llama-rs 或 API（AI 对话）

sysinfo（系统监控）

notify（文件变化）

5.4 通信

tokio::net::TcpListener

或 Windows Named Pipe

cargo-wix / Inno Setup 打包成 .exe
